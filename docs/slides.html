01System Architecture

Linux is layered: your apps ask the shell & libraries, which ask the kernel, which drives the hardware.

👤 User applications browsers, servers, your scripts
🐚 Shell & utilities bash, coreutils (ls, cp, grep…)
📚 Libraries & system calls glibc — the bridge to the kernel
🧠 Kernel processes, memory, filesystems, drivers
🔌 Hardware CPU, RAM, disks, network

02Filesystem Hierarchy

One tree from / — no drive letters. Devices are mounted into it (FHS standard).

/  root
/bin/boot/dev /etc/home/lib /media/mnt/opt /proc/root/sbin /srv/sys/tmp /usr/var
/usr — user programs
/bin/include/lib /local/sbin/src
/var — variable data
/cache/lib/log /mail/spool/tmp
root top-level dirs sub-directories

03Anatomy of Permissions

Reading -rwxr-xr-- : a type bit + three triplets. Each triplet is a number (r=4, w=2, x=1).

So -rwxr-xr-- = chmod 754. Common ones: 755 executables, 644 normal files.

04The Boot Process

From power button to login prompt, in five stages.

1 · Firmware
BIOS/UEFI inits hardware, finds a boot device
→
2 · Bootloader
GRUB loads the kernel + initramfs into memory
→
3 · Kernel
Inits drivers, mounts root FS, starts PID 1
→
4 · systemd
PID 1 starts services to reach the target
→
5 · Login
getty / display manager prompts you in

05Streams & Pipes

Every process has three streams. A | pipe wires one program's output into the next one's input.

stdin (0)
input →
process
→ stdout (1) normal output
→ stderr (2) errors

cat access.log | grep 500 | wc -l — chain small tools to answer big questions.

06Processes & Signals

Processes form a tree from PID 1 (systemd). You control them by sending signals.

Signal№Effect
SIGTERM15Politely ask to stop — the default of kill
SIGKILL9Force-kill, can't be caught — kill -9
SIGINT2Interrupt from keyboard — Ctrl+C
SIGHUP1Hangup — often means "reload config"
SIGSTOP/CONT19/18Pause / resume a process
